# গল্প বড় করা — গাইড

**লক্ষ্য:** গল্পগুলো এমন বড় হোক যেন ইউজার রিয়াল লাইফের উপন্যাস/লিজারের মতো মজা পায় এবং নিজেকে গল্পের ভিতরে পুরোভাবে ডুবিয়ে রাখতে পারে। কয়েকটা চয়েসে যেন শেষ না হয়।

---

## ১. এখনকার সমস্যা

- অনেক গল্প **২–৪টা নোড** (দৃশ্য) আর **২–৩টা এন্ডিং** দিয়েই শেষ হয়ে যাচ্ছে।
- ইউজার ৩–৪টা চয়েস দিয়েই গল্প শেষ দেখে ফেলছে — **immersion** কম।

## ২. গল্প বড় করার নিয়ম

### ক) নোড সংখ্যা বাড়ান

- **ছোট গল্প:** ৪–৬ নোড, ২–৩ এন্ডিং।
- **মাঝারি (টার্গেট):** ১০–১৫ নোড, ৪–৬ এন্ডিং।
- **বড় / উপন্যাস-সদৃশ:** ২০+ নোড, ৬–১০ এন্ডিং।

প্রতি নোড = একটা দৃশ্য বা একটা বড় মুহূর্ত। বেশি নোড = বেশি পড়ার ও চয়েস নেওয়ার সুযোগ।

### খ) গভীরতা (depth) বাড়ান

- **শ্যালো:** start → এক নোড → ending (খুব তাড়াতাড়ি শেষ)।
- **গভীর:** start → কয়েকটা মাঝের দৃশ্য → আবার চয়েস → আরও দৃশ্য → তারপর ending।

মাঝে ২–৩ লেয়ার চয়েস রাখুন; শুধু এক লেয়ার চয়েসে এন্ডিং এ যাবেন না।

### গ) প্রতিটি নোডে কন্টেন্ট লম্বা করুন

- শুধু ২–৩ বাক্য না লিখে **একটা দৃশ্য** বানান: পরিবেশ, চরিত্রের কথা/আচরণ, মood।
- বাংলা গল্প হলে ডায়ালগ + ন্যারেশন দিয়ে একটু বিস্তার দিন যেন রিডার গল্পের ভিতরে থাকতে পারে।

### ঘ) একাধিক পথ (branch) রাখুন

- একই start থেকে ভিন্ন ভিন্ন শাখা (কথা বলল / চুপ থাকল / বেরিয়ে গেল ইত্যাদি)।
- প্রতিটি শাখায় আলাদা মাঝের নোড, তারপর আলাদা বা shared ending।
- এতে এক গল্প অনেকবার খেললে ভিন্ন ভিন্ন অভিজ্ঞতা হবে।

### ঙ) এন্ডিং বেশি রাখুন

- কমপক্ষে ৪–৬ রকম শেষ (মিষ্টি, দুঃখ, মজা, রহস্য, শান্ত, অসম্পূর্ণ ইত্যাদি)।
- ভিন্ন চয়েস কম্বিনেশন যেন ভিন্ন ending এ গিয়ে পৌঁছায়।

---

## ৩. টেকনিক্যালি কী করবেন (Supabase)

- **`story_nodes`** টেবিলে নতুন row যোগ করুন: প্রতিটি দৃশ্য = এক নোড (অনন্য `node_key`)।
- প্রতিটি নোডের **`choices`** JSON এ `next_node` দিয়ে পরের নোডের key লিখুন। একটা চয়েস = একটা সম্ভাব্য পরবর্তী নোড।
- গল্প বড় করলে: মাঝের নোডগুলোতে `is_ending = false`, শুধু শেষ দৃশ্যগুলোতে `is_ending = true` দিন।
- গল্পের বর্ণনা লম্বা করতে প্রতিটি নোডের **`content`** ফিল্ডে বেশি টেক্সট দিন।

কোন নোড থেকে কোন নোডে যাবে সেটা একটা কাগজে বা ডকুমেন্টে ম্যাপ করে নিলে SQL লিখতে সুবিধা হবে।

## ৪. উদাহরণ ম্যাপ (ক্যাফে গল্প বড় করা)

**ছোট (আগের):**  
start → conversation / silent_moment → ending (৩–৪ নোডেই শেষ)

**বড় (টার্গেট):**  
- start → conversation / silent_moment  
- conversation → evening_deepens → moment_of_truth / light_moment → বিভিন্ন ending  
- silent_moment → by_the_window → ending_quiet  

এভাবে মাঝে ২–৩ লেয়ার যোগ করলে নোড সংখ্যা ও পড়ার পরিমাণ দুটোই বাড়ে।

**এই রিপোতে উদাহরণ:** `add_story_romance_LONG.sql` — একই ক্যাফে গল্প, কিন্তু বেশি নোড (start → conversation → evening_deepens → moment_of_truth → ending; অথবা silent_moment → by_the_window → ending)। Supabase এ এই ফাইল রান করলে বড় গল্পটা সিড হয়ে যাবে।

- [ ] প্রতিটি নোডে কন্টেন্ট একটু লম্বা (দৃশ্য + ডায়ালগ/ন্যারেশন)
